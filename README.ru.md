# DocuMind — Интеллектуальная система вопросов и ответов на основе RAG

> Production-ready RAG-система, написанная с нуля — без высокоуровневых абстракций вроде LangChain.  
> Загружайте PDF, TXT или DOCX файлы и задавайте вопросы на естественном языке.

---

## Что такое RAG и зачем он нужен

**RAG (Retrieval-Augmented Generation)** — это архитектурный паттерн, который комбинирует поиск по документам с языковой моделью для генерации точных ответов.

### Проблема, которую решает RAG

Большие языковые модели (LLM) обладают двумя фундаментальными ограничениями:

1. **Знания ограничены датой обучения** — модель не знает о ваших внутренних документах, последних событиях или корпоративной базе знаний.
2. **Галлюцинации** — LLM могут уверенно выдавать ложную информацию, "додумывая" факты, которых не знает.

RAG решает оба эти ограничения: вместо того чтобы полагаться на "память" модели, система сначала **находит** релевантные фрагменты в ваших документах, а затем передаёт их в модель как контекст.

### Как это работает: две фазы

**Фаза индексирования (выполняется один раз):**
```
Документ → Разбивка на чанки → Векторизация → Сохранение в векторную БД
```

1. Документ разбивается на небольшие фрагменты (чанки) — по границам предложений или фиксированного размера
2. Каждый чанк преобразуется в числовой вектор (embedding) через нейросеть-энкодер
3. Вектора сохраняются в векторную базу данных (ChromaDB)

**Фаза ответа на вопрос:**
```
Вопрос → Векторизация → Поиск похожих чанков → LLM с контекстом → Ответ
```

1. Вопрос пользователя тоже превращается в вектор
2. В базе данных ищутся чанки с наибольшим косинусным сходством (семантически похожие)
3. Найденные фрагменты + вопрос отправляются в LLM
4. Модель отвечает, опираясь ТОЛЬКО на предоставленный контекст

### Почему гибридный поиск лучше

В этом проекте реализован **гибридный ретривер**:

- **Семантический поиск** (ChromaDB + all-MiniLM-L6-v2) — находит тематически похожие тексты даже при разных словах
- **BM25** (keyword search) — классический алгоритм поиска по ключевым словам, работает лучше для точных терминов и имён собственных
- **RRF (Reciprocal Rank Fusion)** — алгоритм слияния, который объединяет результаты обоих методов

Это устраняет слабые стороны каждого подхода в отдельности.

### Ключевые технические решения

| Компонент | Выбор | Причина |
|-----------|-------|---------|
| Эмбеддинги | `sentence-transformers/all-MiniLM-L6-v2` | Быстро на CPU, высокое качество |
| Векторная БД | ChromaDB | Персистентная, не требует инфраструктуры |
| Поиск | Семантика + BM25 через **RRF** | Лучшее из обоих миров — плотный + разреженный поиск |
| Генерация | HF `InferenceClient` (chat API) | Не нужна локальная GPU, есть бесплатный тир |
| Backend | FastAPI | Async, авто-документация, production-готов |
| Frontend | Streamlit | Быстрый прототип с загрузкой файлов |

## Быстрый старт

### 1. Установка зависимостей

```bash
pip install -r requirements.txt
```

### 2. Получить бесплатный токен HuggingFace

Зайдите на [https://huggingface.co/settings/tokens](https://huggingface.co/settings/tokens) и создайте токен с правами **Read**.

```bash
cp .env.example .env
# Откройте .env и добавьте ваш HF_TOKEN
```

### 3. Запуск Streamlit UI (рекомендуется)

На Windows (PowerShell):
```powershell
$env:HF_TOKEN="hf_ваш_токен_здесь"
python -m streamlit run ui/streamlit_app.py
```

На Linux/Mac:
```bash
HF_TOKEN=hf_ваш_токен_здесь streamlit run ui/streamlit_app.py
```

Откройте браузер: `http://localhost:8501`

### 4. Поддерживаемые LLM модели

| Модель | Описание |
|--------|----------|
| `Qwen/Qwen2.5-7B-Instruct` | **По умолчанию** — быстрая и точная |
| `meta-llama/Meta-Llama-3.1-8B-Instruct` | Отлично следует инструкциям |
| `HuggingFaceH4/zephyr-7b-beta` | Лёгкая альтернатива |
| `microsoft/Phi-3-mini-4k-instruct` | Компактная, минимальная задержка |

Все модели работают через [HuggingFace Inference API](https://huggingface.co/docs/api-inference) — локальная GPU не нужна.

---

### 5. Запуск FastAPI backend

```powershell
$env:HF_TOKEN="hf_ваш_токен_здесь"
uvicorn api.main:app --reload
```

Интерактивная документация API: `http://localhost:8000/docs`

## Использование через Streamlit UI

1. В боковой панели введите ваш HuggingFace токен
2. Выберите модель LLM (по умолчанию Mistral-7B-Instruct)
3. Нажмите **Initialize Pipeline** — загрузится модель эмбеддингов
4. Загрузите PDF, TXT или DOCX файл
5. Нажмите **Ingest Documents** — документ будет проиндексирован
6. В чате задавайте вопросы по документу

## API Reference

### `POST /ingest/file`
Загрузить файл для индексирования.

```bash
curl -X POST http://localhost:8000/ingest/file \
  -F "file=@your_document.pdf"
```

### `POST /query`
Задать вопрос по загруженным документам.

```bash
curl -X POST http://localhost:8000/query \
  -H "Content-Type: application/json" \
  -d '{"question": "О чём этот документ?"}'
```

### `GET /health`
Проверить статус пайплайна и количество документов.
